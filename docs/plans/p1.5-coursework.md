# Phase P1.5 — Coursework Engine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Study guides and fully-cited practice exams generated from real course materials, quiz mode with grading and a weak-topic review queue, and syllabus-deadline import as suggestion rows.

**Architecture:** `backend/study/` package. Agent generation is dependency-injected (a `Generator = Callable[[prompt], Awaitable[str]]`) so all validation/rendering logic is unit-testable with fakes; the real generator wraps claude-agent-sdk `query()`. Citation enforcement (I6) is pure logic: a question survives only if its citation resolves to a chunk of the course corpus. Study writes create NEW files under `15-Courses/<C>/study/` only (I1 exemption); syllabus import only inserts suggestion rows (kind=task).

## Tasks

### T1 — corpus.py + db schema
`course_corpus(index, course) -> list[chunk]` (chroma `where={"course": ...}`); `courses(vault) -> [CourseInfo{code,title,path,materials,notes}]` from `15-Courses/*/course.md`. New tables: `exams(id, course, title, created_at, questions_json)`, `attempts(id, exam_id, created_at, score, total, answers_json, weak_topics)`.

### T2 — practice_exam.py (core, TDD)
`build_exam(course, corpus, raw_json) -> Exam` — parse strict JSON `{questions:[{q,type(mcq|short|problem),options?,answer,explanation,citation{path,page?,slide?,quote}}]}`; DROP any question whose `citation.quote` (normalized) is not a substring of the cited chunk's text or whose path isn't in the corpus (I6). `render_exam_md/key_md`. `generate_practice_exam(...)` orchestrates: prompt from corpus sample → generator → build → write exam+key under `study/` → insert db row. Tests: uncited/misquoted question dropped; cited kept; renders; db row.

### T3 — study_guide.py
`generate_study_guide(course, scope)` — corpus-scoped prompt (outline, key concepts w/ citations, worked examples, gap list comparing materials vs notes chunks) → write `study/guide-<scope>-<date>.md`. Gap list computed in code: material topics (headings/slides) with no similar notes chunk. Test: file written under study/ only; gap list contains material-only topic.

### T4 — grader.py
`grade_attempt(conn, vault, exam_id, answers) -> AttemptResult{score,total,feedback[],weak_topics[]}` — mcq exact match (case-insensitive letter/text), short/problem via normalized containment; weak topics = citations' headings/titles of missed questions; append to `study/review-queue.md`; insert attempts row. Tests: scoring, review-queue append, attempt row.

### T5 — syllabus.py
`parse_syllabus(file, course, conn)` — extract text (rag.extract), find dated items (regex `\b(\d{4}-\d{2}-\d{2}|(Jan|...)\s+\d{1,2})` + line context, keywords exam/quiz/due/assignment/project) → suggestion rows kind=task with lead-time note. Never writes tasks (I1). Test: fixture syllabus → ≥2 suggestion rows with dates; no vault writes.

### T6 — API + web/study
Endpoints: `GET /api/study/courses`, `POST /api/study/upload` (multipart → materials/, then index.upsert), `POST /api/study/guide {course,scope}`, `POST /api/study/exam {course,topics?,n,difficulty}`, `GET /api/study/exams?course=`, `GET /api/study/exams/{id}` (questions sans answers), `POST /api/study/exams/{id}/attempt {answers}`. Generator injectable app-wide (like chat_runner). Web study page: course cards, upload, generate buttons, exam list, quiz mode (one question at a time → grade → show citations + weak topics). Tests: upload lands in materials/; exam endpoint hides answers; attempt returns score.

### T7 — exit criteria
pytest all green; manual: real lecture PDF (CM8 Plate Tectonics) → 10-question exam, every question cites a real page; quiz round-trip in API; npm build; merge/push; BUILD_STATE/LOG.
